## 第一阶段

### Q1. 磁盘分区信息怎么获取

`MakeDriverInfo()` 函数（`RemoteCtrl/RemoteCtrl.cpp`）：

用 `_chdrive(i)` 遍历 1~26（对应 A~Z 盘），返回 0 表示该盘符存在，将存在的盘符拼成逗号分隔的字符串（如 `"C,D,E"`），封包成 Cmd=1 的 CPacket 发给客户端。

```cpp
for (int i{1}; i <= 26; ++i) {
    if (_chdrive(i) == 0) {
        if (result.size() > 0) result += ',';
        result += 'A' + i - 1;
    }
}
CPacket pack{1, (const BYTE*)result.c_str(), result.size()};
CServerSocket::getInstance()->Send(pack);
```

### Q2. 被控端怎么获取文件和目录信息的

`MakeDirectorytInfo()` 函数（`RemoteCtrl/RemoteCtrl.cpp`）：

1. 从收到的包中取出路径字符串，用 `_chdir()` 切换到该目录
2. 用 `_findfirst("*", &fdata)` + `_findnext()` 遍历目录下所有文件/目录
3. 每个文件填充一个 `FILEINFO` 结构体（含 `IsDirectory`、`szFileName`、`HasNext=TRUE`），封成 Cmd=2 的 CPacket 逐条发送
4. 遍历完后发一个 `HasNext=FALSE` 的空 FILEINFO 表示结束

```cpp
_finddata_t fdata;
intptr_t hfind{0};
hfind = _findfirst("*", &fdata);
do {
    FILEINFO finfo;
    finfo.IsDirectory = (fdata.attrib & _A_SUBDIR) != 0;
    memcpy(finfo.szFileName, fdata.name, strlen(fdata.name));
    CPacket pack{2, (BYTE*)&finfo, sizeof(finfo)};
    CServerSocket::getInstance()->Send(pack);
} while (!_findnext(hfind, &fdata));
// 发送结束标志
FILEINFO finfo;
finfo.HasNext = FALSE;
CPacket pack{2, (BYTE*)&finfo, sizeof(finfo)};
CServerSocket::getInstance()->Send(pack);
```

### Q3. 鼠标操作是怎么处理的

`MouseEvent()` 函数（`RemoteCtrl/RemoteCtrl.cpp`）：

1. 从包中解析 `MOUSEEV` 结构体（含 `nAction`：单击/双击/按下/放开，`nButton`：左/右/中键，`ptXY`：坐标）
2. 先调 `SetCursorPos()` 移动鼠标到目标坐标
3. 根据 `nButton`（0=左/1=右/2=中）和 `nAction`（0=单击/1=双击/2=按下/3=放开）组合成 `nFlags`，switch 分发调用 `mouse_event()` API 模拟对应的鼠标事件

```cpp
// MOUSEEV 结构体
typedef struct MouseEvent {
    WORD nAction;   // 单击、移动、双击...
    WORD nButton;   // 左键、右键、中键
    POINT ptXY;     // 坐标
} MOUSEEV, *PMOUSEEV;

SetCursorPos(mouse.ptXY.x, mouse.ptXY.y);
// 以左键单击为例
mouse_event(MOUSEEVENTF_LEFTDOWN, 0, 0, 0, GetMessageExtraInfo());
mouse_event(MOUSEEVENTF_LEFTUP,   0, 0, 0, GetMessageExtraInfo());
```

### Q4. 被控端的屏幕是怎么获取的

`SendScreen()` 函数（`RemoteCtrl/RemoteCtrl.cpp`）：

1. `GetDC(NULL)` 获取屏幕 DC，读取分辨率（`HORZRES`/`VERTRES`）和色深（`BITSPIXEL`）
2. 创建 `CImage`（GDI+），用 `BitBlt()` 将屏幕内容复制到 CImage
3. `GlobalAlloc()` 分配内存，`CreateStreamOnHGlobal()` 创建内存流
4. 调 `screen.Save(pStream, Gdiplus::ImageFormatPNG)` 将截图以 PNG 格式写入内存流
5. `GlobalLock()` 拿到原始字节，封成 Cmd=6 的 CPacket 发送

```cpp
CImage screen;
HDC hScreen = ::GetDC(NULL);
int nWidth  = GetDeviceCaps(hScreen, HORZRES);
int nHeight = GetDeviceCaps(hScreen, VERTRES);
screen.Create(nWidth, nHeight, nBitPerPixel);
BitBlt(screen.GetDC(), 0, 0, nWidth, nHeight, hScreen, 0, 0, SRCCOPY);

HGLOBAL hMem = GlobalAlloc(GMEM_MOVEABLE, 0);
IStream* pStream = nullptr;
CreateStreamOnHGlobal(hMem, TRUE, &pStream);
screen.Save(pStream, Gdiplus::ImageFormatPNG);

PBYTE pData = (PBYTE)GlobalLock(hMem);
SIZE_T nSize = GlobalSize(hMem);
CPacket pack(6, pData, nSize);
CServerSocket::getInstance()->Send(pack);
```

### Q5. 锁机和解锁怎么实现

**锁机** `LockMachine()`（`RemoteCtrl/RemoteCtrl.cpp`）：

用 `_beginthreadex()` 启动新线程 `threadLockDlg`，线程中：
- 创建全屏无边框对话框，`SetWindowPos(TOPMOST)` 置顶
- `ShowCursor(false)` 隐藏鼠标光标
- `ShowWindow(FindWindow("Shell_TrayWnd",...), SW_HIDE)` 隐藏任务栏
- `ClipCursor()` 限制鼠标范围在对话框内
- 自己跑消息循环，等待 `WM_KEYDOWN` + 键值 `0x41`（'A'）才退出

**解锁** `UnlockMachine()`（`RemoteCtrl/RemoteCtrl.cpp`）：

用 `PostThreadMessage(threadid, WM_KEYDOWN, 0x41, 0x1E0001)` 向锁机线程发送 'A' 键消息，触发其退出，线程清理时恢复光标、显示任务栏。

```cpp
// 锁机
_beginthreadex(NULL, 0, threadLockDlg, NULL, 0, &threadid);

// 解锁
PostThreadMessage(threadid, WM_KEYDOWN, 0x41, 0x1E0001);
```

### Q6. CPacket 包是怎么封装的，有没有考虑内存对齐

**包格式**（`RemoteCtrl/ServerSocket.h` / `RemoteClient/ClientSocket.h`）：

```
[sHead 2B: 0xFEFF][nLength 4B][sCmd 2B][Data N B][sSum 2B]
```

| 字段 | 大小 | 说明 |
|------|------|------|
| sHead | 2 字节 | 固定头 `0xFEFF`，用于帧同步 |
| nLength | 4 字节 | `sizeof(Data) + 4`（含 sCmd 和 sSum，不含 sHead/nLength 自身） |
| sCmd | 2 字节 | 命令编号 |
| Data | N 字节 | 载荷数据 |
| sSum | 2 字节 | Data 各字节累加校验和 |

序列化时用指针手动写入各字段，不依赖结构体内存布局：

```cpp
BYTE* pData = (BYTE*)strOut.data();
*(WORD*)pData  = sHead;    pData += 2;
*(DWORD*)pData = nLength;  pData += 4;
*(WORD*)pData  = sCmd;     pData += 2;
memcpy(pData, strData.c_str(), strData.size());
pData += strData.size();
*(WORD*)pData  = sSum;
```

**关于内存对齐**：CPacket 的序列化是手动字节写入，**不依赖结构体对齐**，协议格式固定为小端序，是正确的做法。

但 `FILEINFO` 和 `MOUSEEV` 结构体是直接 `memcpy` 整体收发，这两个结构体**没有加 `#pragma pack`**，依赖编译器默认对齐。这是一个潜在隐患：若客户端与服务端编译设置不同，结构体布局可能不一致，导致字段错位。严谨做法应加 `#pragma pack(1)` 或手动按字节解析。
